REST API
Backend Endpoints
POST
/api/analyze
Analyze a UPI transaction. Sends VPA, amount, type, remarks through the ML fraud engine. Returns risk_score, verdict, confidence, and flags.
GET
/api/transactions
Retrieve recent transaction history from SQLite DB. Supports ?limit=50 and ?verdict=fraud|suspicious|safe filters.
GET
/api/stats
Aggregated dashboard metrics: total scanned, frauds blocked, amount protected, avg risk score, and 12-hour hourly chart data.
GET
/api/vpa/lookup?vpa=
Look up a VPA against the verified registry. Returns entity name, verified status, and historical fraud count.
POST
/api/report
File a fraud report for a transaction ID. Packages evidence and submits to RBI CFCFRMS portal automatically.
GET
/api/stream
Server-Sent Events stream. Connect to receive live transaction events as they are processed by the fraud engine.
Technology
How ShieldPay Works
01

RandomForest ML Model

120-tree ensemble classifier trained on synthetic fraud patterns. Features include VPA entropy, amount log-scale, time-risk, type-risk, NLP note score, and mule-pattern detection.

02

Hybrid Scoring Engine

ML probability (60%) combined with deterministic rule engine (40%). Hard overrides for known fraud VPAs and verified merchants ensure zero false negatives on critical patterns.

03

NLP Remarks Analysis

Pattern matching against 30+ social engineering keywords across transaction notes. Multi-keyword hits escalate risk score exponentially.

04

SQLite Transaction Store

All transactions, fraud reports, and VPA registry data persisted in SQLite. Supports full query history, filtering by verdict, and statistical aggregation.

05

SSE Live Stream

Server-Sent Events push each analyzed transaction to connected clients in real time. Background simulator generates realistic mixed traffic for live demo.

06

AMD RYZEN AI Ready

Fraud engine designed for deployment on AMD RYZEN AI NPU via ONNX export. Sub-80ms inference on-device — no data leaves the endpoint.
